# Improve Command

Enhance an existing working feature.

## Input
$ARGUMENTS

Provide a file path or feature name to improve.

## Instructions

1. **Review the current implementation**
   - Read and understand the existing code
   - Identify how it currently works

2. **Analyze improvement opportunities** across these dimensions:

   ### Performance
   - Reduce unnecessary re-renders
   - Optimize database queries
   - Add caching where beneficial

   ### Readability
   - Clearer naming conventions
   - Better code structure
   - Simplify complex logic

   ### UX
   - Loading states
   - Error messages
   - Edge case handling

   ### Accessibility
   - ARIA attributes
   - Keyboard navigation
   - Screen reader support

   ### Error Handling
   - Graceful failures
   - User feedback
   - Logging for debugging

3. **Propose changes with rationale**
   - Explain why each improvement matters
   - Prioritize by impact

4. **Implement incrementally**
   - One improvement at a time
   - Verify each change works before moving on
